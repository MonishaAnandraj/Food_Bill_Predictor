# Food Bill Predictor 🍔

A Machine Learning-powered web application that predicts the total bill for food delivery based on various factors like restaurant, cuisine, number of items, and customer demographics.

## 🚀 Overview
This project consists of a full-stack data science workflow:
1.  **Data Analysis & Modeling**: A Jupyter Notebook (`Food_delivery_bill_prediction.ipynb`) that handles data cleaning, handling missing values, exploratory data analysis, and training a regression model.
2.  **Web Application**: A Streamlit-based interface (`app.py`) that allows users to input order details and receive real-time bill predictions.

## ✨ Features
- **Real-time Prediction**: Instantly calculates the expected total bill using a pre-trained `model.pkl`.
- **Interactive UI**: User-friendly sidebar for selecting restaurant names, cuisine types, and meal times.
- **KPI Dashboard**: Displays key metrics such as the base price of items and the amount saved via discounts.
- **Comprehensive Inputs**: Considers factors like:
    - Restaurant (e.g., Spice Garden, Pizza Palace, Sushi Stop).
    - Cuisine (Indian, Italian, Japanese, etc.).
    - Delivery distance and customer demographics.

## 🛠️ Tech Stack
- **Language**: Python
- **Libraries**: Pandas, Scikit-learn, Joblib, Streamlit
- **Deployment**: Streamlit

2. **Install dependencies:**

## 📊 Model Details
The model was trained on a dataset containing 200 entries with 14 original columns. The preprocessing pipeline includes:
- Handling missing values using mean/mode imputation.
- Filtering outliers (e.g., removing negative item prices).
- Feature engineering for categorical variables like `restaurant_name` and `cuisine_type`.
- Developed a `Linear Regression model` to predict food delivery bills.
